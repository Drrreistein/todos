# 产品规格：极简待办清单 (Minimal Todo)

## 产品定位

一个追求极致克制的待办清单 Web App。设计哲学：去掉一切多余的元素，只保留核心交互。
灵感来自 Things 3、Linear、Notion 的极简美学，但更轻量。

## 核心用户故事

1. 用户打开页面，看到当前所有待办事项（本地存储持久化）
2. 用户在输入框中输入待办内容，按 Enter 或点击添加按钮，新条目出现在列表顶部
3. 用户点击条目左侧复选框，该条目标记为完成（文字带删除线，颜色变灰）
4. 用户点击已完成条目的删除图标，条目从列表中移除
5. 用户可以看到当前待完成条目数量
6. 用户可以一键清除所有已完成条目

## 技术栈

- **框架**: React 18 + Vite + TypeScript
- **样式**: Tailwind CSS v4（无 UI 库，全手写组件）
- **状态**: useState + useReducer（纯前端，无后端）
- **持久化**: localStorage
- **测试**: Vitest + Playwright

## 视觉设计规范

### 极简设计原则
- **色彩**：仅使用黑、白、灰三色系。无彩色，无渐变，无阴影滥用
- **字体**：系统字体栈（-apple-system, Segoe UI, sans-serif），字重区分层级
- **间距**：宽松留白，每个元素都有充足的呼吸空间
- **动效**：仅保留必要的微交互（复选框打勾动画、条目出现/消失过渡）
- **禁止项**：紫色渐变、彩色卡片、多层阴影、花哨图标、圆角过大

### 视觉层级
- 页面标题：大字号，轻字重（font-light），顶部居中
- 待完成数量：小号辅助文字，灰色
- 输入区域：细边框，focus 时边框变黑
- 待办条目：无卡片包裹，纯列表形式，hover 时背景微变
- 已完成条目：删除线 + 浅灰色，视觉上退到背景

## 功能清单（按优先级）

### P0 核心功能
- 添加待办（Enter 键 + 按钮）
- 完成/取消完成切换
- 删除单条（仅已完成条目可直接删除，未完成条目 hover 显示删除）
- localStorage 持久化
- 待完成数量显示

### P1 增强功能
- 清除所有已完成
- 空状态提示（列表为空时）
- 输入框空内容校验（禁止添加空白待办）

### P2 可选功能
- 深色模式切换（可选）

## 验收标准

每个功能必须通过端到端测试才能标记完成，不接受仅靠代码审查的完工声明。
